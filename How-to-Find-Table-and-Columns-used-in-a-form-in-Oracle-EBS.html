How to Find the Table and Columns used in a Form in Oracle EBS

By DarkSoul  -  December 05, 2024

Have you ever stared at an Oracle EBS form and wondered, What table is this pulling data from? What columns are being updated? As developers or technical consultants, we often face this question while customizing reports, creating integrations, or debugging issues.

Fear not! Oracle EBS has built-in tools and techniques to unravel this mystery, and I’m here to guide you through the process. By the end of this blog, you’ll have the confidence to trace the database objects behind any form.

Step 1: Identify the Form

  • Open a form in Oracle EBS (say Users form).
  • Note the name or purpose of the form, as this will help you narrow down the relevant tables and columns.

Step 2: Check the Record History

Oracle EBS has a nifty feature called Record History, which can directly hint at the table behind the form.

  • Navigate to Help > Record History from the menu.
  • A pop-up window will appear, showing the base table and primary key columns associated with the current record in the form.

Pro Tip: This step is often the quickest way to identify the main table, so always check here first.

Step 3: Enable Diagnostics Mode

Sometimes, the Record History feature might not give you all the details you need. That’s when Diagnostics mode comes to the rescue.

  • Go to Help > Diagnostics > Examine.
  • If prompted, enter the Apps password. This allows you to inspect the block, field, and associated database objects for the form.
  • In the pop-up window:
    • Block: Select the block associated with the form.
    • Field: Select the field you’re investigating.

The block and field information will usually be tied to a table and column in the database. Note these details down.

Note: Ensure you have the necessary permissions to use Diagnostics mode.

Step 4: Review the Table Schema in SQL Developer

Now that you have some table and field names, it’s time to dig deeper using an SQL tool like SQL Developer or TOAD.

Run the below query to explore the table and its columns:

  SELECT column_name, data_type
  FROM all_tab_columns
  WHERE table_name = 'table_name'
  ORDER BY column_id;

Replace 'table_name' with the table identified in steps 2 or 3.

Pro Tip: If the table name includes a prefix (like AP_, AR_, etc.), it’s likely tied to a specific module (e.g., Accounts Payable, Accounts Receivable). This can guide your exploration.

Step 5: Check the View Definitions (if Applicable)

Forms often query views rather than direct tables. If you suspect a view is involved:

Run this query to get the view definition:

  SELECT text
  FROM all_views
  WHERE view_name = 'view_name';

Analyze the query within the view to trace the underlying tables.

Note: Views often combine multiple tables or include calculated fields, so understanding the view logic is key.

Wrapping Up

With these steps, you’re equipped to confidently trace the backend structure of any Oracle EBS form. Whether you’re analyzing the Record History, leveraging Diagnostics mode, or diving into SQL queries, there’s always a way to uncover the mystery behind the scenes.

Remember, understanding the database structure not only helps with debugging but also empowers you to build better customizations and reports.

So, the next time you’re staring at an EBS form, don’t just wonder. Go explore!
