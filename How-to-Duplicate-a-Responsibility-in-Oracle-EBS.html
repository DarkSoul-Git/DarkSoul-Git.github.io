How to Duplicate a Responsibility in Oracle Apps R12 EBS

By DarkSoul  -  November 23, 2024

Managing responsibilities efficiently in Oracle E-Business Suite (EBS) is a key part of system administration. Sometimes, you may need to create a new responsibility that’s similar to an existing one—perhaps for a different team, user role, or department. Instead of manually recreating all the details, you can use shortcuts and tools within Oracle EBS to duplicate responsibilities quickly and accurately.

This guide walks you through a step-by-step process to duplicate a responsibility, highlighting useful shortcuts like Ctrl + Down Arrow and Shift + F5, and providing tips for saving time.

Why This Method Works?

Using Ctrl + Down Arrow (or the New icon) followed by Shift + F5 allows you to quickly duplicate the responsibility’s data, field by field. While Shift + F5 duplicates only the field where the cursor is located, it provides an easy shortcut to replicate the core data of the responsibility. After that, you can manually copy the remaining details to complete the new record.

Before starting, make sure you have System Administrator access in Oracle EBS.

Step 1: Navigate to the Define Responsibility Form

Log in with System Administrator access and navigate to:

  • System Administrator > Security > Responsibility > Define

This form is where all existing responsibilities are listed, and where you can query, edit, or create new responsibilities.

Step 2: Query the Responsibility You Want to Duplicate

Here’s how to find the responsibility you want to copy:

  • First, press F11 when the form gets opened. It will get the form to query mode.
  • In the fields, enter the Responsibility Name or Key.
  • Press Ctrl + F11 to query the responsibility.

The queried responsibility will appear in the form, ready for duplication.

Step 3: Create a New Responsibility

Once the existing responsibility is displayed, follow these steps to start creating a new one:

  • Press Ctrl + Down Arrow, or click the New icon (a blank page icon).After clicking the new icon, the form will clear and prepare a new responsibility record to fill.
  • Now, enter the Responsibility Name, Responsibility Key and Description. Since the Responsibility Name and Key are unique for each responsibility, you shouldn't duplicate these fields

Step 4: Duplicate Data Using Shift + F5

To quickly copy data from the original responsibility into the new one:

  • Move the cursor to the field you want to duplicate (e.g., Application).
  • Press Shift + F5 to copy the data from the original responsibility for the selected field.

Note: The Shift + F5 shortcut only duplicates the data in the field where the cursor is positioned. It does not copy the entire record. You’ll need to manually copy the remaining data (like menus, request groups, etc.) to the new responsibility.

Step 5: Manually Copy the Remaining Data

Since Shift + F5 duplicates only one field at a time, manually copy the remaining fields for your new responsibility:

  • Menu: Assign the same or a different menu to the new responsibility.
  • Request Group: Set the same or a new request group.
  • Update other fields, like Data Group, as needed.

Step 6: Save the New Responsibility

Once all fields are updated, click the Save icon (a diskette icon) to save the new responsibility.

Step 7: Assign the Responsibility to a User

Now that the responsibility is created, assign it to a user:

  • Navigate to Security > User > Define.
  • Search for the user, and in the Responsibilities tab, add the new responsibility.

Step 8: Test the Responsibility

Log in as the assigned user (or have them do so) and verify:

  • Are the correct menu options available?
  • Can the user access assigned reports or concurrent programs?
  • Do restrictions, like menu exclusions, function as expected?

That’s It!

Congratulations! You’ve successfully duplicated a responsibility in Oracle EBS. This approach saves time and ensures consistency when setting up user access. If you run into any issues or have questions, feel free to ask. Happy configuring!

Keyboard Shortcuts

▸ F11 - to get into Query mode in oracle forms.

▸ Ctrl + F11 - to query/search for records in forms.

▸ Ctrl + Down arrow - to create a new record.

▸ Shift + F5 - to Duplicate the data for the selected field..
